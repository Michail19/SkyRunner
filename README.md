# SkyRunner

**SkyRunner** — учебная 3D-игра на Unity в жанре survival/arena runner. Игрок появляется на процедурно собранной арене из плиток, собирает предметы, избегает падающих участков платформы и уклоняется от ботов, которые пытаются столкнуть его за пределы карты.

Проект ориентирован на демонстрацию базовых игровых систем Unity: управление персонажем от первого/третьего лица, процедурная генерация арены, разрушаемые плитки, простая логика противников, сбор предметов, условия победы/поражения, меню, настройки сложности и звука.

## Кратко о проекте

Цель проекта — создать небольшую, но завершённую survival-игру, в которой игроку нужно удержаться на арене, собрать необходимые предметы и добраться до выхода.

Ключевая идея: арена небезопасна. Плитки постепенно исчезают, количество угроз увеличивается, а противники стараются вытолкнуть игрока за пределы уровня. Игрок должен постоянно двигаться, выбирать безопасный маршрут и успеть выполнить цель до поражения.

---

## Игровой процесс

Игровой цикл состоит из нескольких этапов:

1. Игрок появляется на центральной безопасной зоне арены.
2. На карте генерируются плитки разных типов: обычные, низкие/водные и каменные.
3. На некоторых плитках размещаются собираемые предметы.
4. Через заданное время случайные плитки начинают предупреждать игрока и затем исчезают.
5. На арене появляются боты, которые движутся к игроку и пытаются его столкнуть.
6. После сбора всех предметов активируется зона выхода.
7. Игрок побеждает, если добирается до выхода.
8. Игрок проигрывает, если падает ниже допустимого уровня.

## Основные возможности

- процедурная генерация островной арены из отдельных плиток;
- разные типы поверхности с разным влиянием на скорость движения;
- защищённая центральная зона старта;
- постепенное разрушение плиток;
- предупреждение перед исчезновением плитки;
- сбор предметов;
- активация выхода после выполнения цели;
- боты-противники, которые преследуют игрока;
- knockback-эффект при атаке бота;
- система сложности: Easy, Normal, Hard;
- настройки чувствительности мыши и громкости;
- меню, пауза, экран победы и поражения;
- звуковые эффекты шагов, сбора предметов, атак, разрушения плиток и окончания игры.

| Действие | Клавиша / действие |
|---|---|
| Движение вперёд | `W` |
| Движение назад | `S` |

| Обзор камеры | Движение мыши |
| Тест knockback в debug-режиме | `K` |

> Управление реализовано через `UnityEngine.InputSystem` внутри `PlayerController`.

## Сложность

В проекте предусмотрены три уровня сложности.

| Сложность | Предметы | Старт разрушения | Время предупреждения | Боты |
|---|---:|---:|---:|---:|
| Easy | 2 | 8 сек. | 2 сек. | меньше и медленнее |
| Normal | 3 | 5 сек. | 1.5 сек. | стандартные параметры |
| Hard | 4 | 3 сек. | 1.1 сек. | больше и быстрее |

Параметры сложности задаются в `GameSettings` и применяются к системам разрушения плиток, спавна ботов и количеству предметов.

## Запуск проекта в Unity

1. Склонировать репозиторий:

```bash
git clone https://github.com/Michail19/SkyRunner.git
cd SkyRunner
git checkout develop
```

2. Открыть проект через Unity Hub.
3. Убедиться, что используется Unity **6000.4.1f1** или совместимая версия.
4. Открыть сцену:

```text
Assets/Scenes/Menu.unity
```

или сразу игровой уровень:

```text
Assets/Scenes/Level.unity
```

5. Нажать `Play`.

---

## Сборка игры

1. Открыть Unity.
2. Перейти в:

```text
File → Build Settings
```

3. Проверить, что в списке сцен есть:

```text
Assets/Scenes/Menu.unity
Assets/Scenes/Level.unity
```

4. Выбрать платформу, например:

```text
Windows, Mac, Linux
```

5. Нажать `Build` или `Build And Run`.

Для отладки производительности рекомендуется включить:

```text
Development Build
Autoconnect Profiler
```

После этого можно открыть:

```text
Window → Analysis → Profiler
```

и проверить, что именно нагружает игру: рендер, скрипты, физика или память.

## Оптимизация

После сборки игра может работать хуже, если включены тяжёлые настройки графики или на сцене слишком много отдельных объектов. Для улучшения производительности рекомендуется выполнить следующие действия.

### 1. Упростить настройки качества

В `Project Settings → Quality` для Standalone можно уменьшить нагрузку:

```text
Shadows: Disable Shadows или Hard Shadows Only
Shadow Distance: 10–20
Shadow Cascades: No Cascades
Anti Aliasing: 0x или 2x
VSync Count: Don't Sync
```

Также можно временно проверить лёгкий профиль:

```csharp
QualitySettings.SetQualityLevel(0);
Application.targetFrameRate = 60;
```

### 2. Уменьшить количество плиток

Для теста можно снизить размер арены:

```csharp
public int gridSize = 12;
```

Если FPS заметно увеличился, основная нагрузка связана с количеством плиток, коллайдеров и рендереров.

### 3. Не создавать копии материалов для каждой плитки

Вместо:

```csharp
tileRenderer.material = targetMaterial;
```

лучше использовать:

```csharp
tileRenderer.sharedMaterial = targetMaterial;
```

Если нужно менять индивидуальные параметры материала, лучше использовать `MaterialPropertyBlock`.

### 4. Ограничить количество ботов

Для слабых ПК можно уменьшить параметры:

```csharp
maxAliveBots = 3;
maxAliveBotsLimit = 5;
spawnInterval = 10f;
```

Также можно увеличить интервал проверки дыр у бота:

```csharp
holeCheckInterval = 1f;
```

### 5. Оптимизировать звуки

`AudioSource.PlayClipAtPoint` создаёт временный объект. Если звуков много, лучше заменить такие вызовы на пул `AudioSource`.

## Известные проблемы и рекомендации

### Прыжок не работает

Проверить следующие пункты:

1. У игрока должен быть корректный `Collider`.
2. Размер `BoxCollider` не должен иметь нулевые значения.
3. У пола/плиток должны быть обычные Collider, не `Is Trigger`.
4. Слой пола должен входить в `groundMask`.
5. В `Rigidbody` не должна быть заморожена позиция по Y.
6. `GamePauseState.IsPaused` должен быть `false`.

Рекомендуемая настройка для игрока:

```text
CapsuleCollider
Radius: 0.3
Height: 2
Direction: Y Axis
```

или для `BoxCollider`:

```text
Size: X = 0.5, Y = 1, Z = 0.5
```

### FPS ниже ожидаемого после build

Проверить:

- профиль качества Standalone;
- тени и Shadow Distance;
- количество плиток на арене;
- количество активных ботов;
- частые вызовы `PlayClipAtPoint`;
- использование `renderer.material` вместо `renderer.sharedMaterial`;
- количество активных источников света и эффектов.

### Игрок проваливается или некорректно стоит на плитках

Проверить:

- размеры Collider у плиток;
- размеры Collider у игрока;
- слой плиток;
- включён ли Collider у плитки после генерации;
- не отключается ли плитка до завершения предупреждения.
